# Optional Labs

The core **Modern C# Type Design** course does not require local setup.

Labs are optional follow-up material for learners who want to run examples, refactor code, or inspect benchmarks locally.

## Lab principles

- Labs should reinforce the guided lessons, not replace them.
- A learner should be able to complete the core course without cloning the repository.
- Setup instructions belong here, not in the first lesson.
- Benchmarks are useful later, but they should not be the first learner experience.

## Future labs

Potential follow-up labs:

1. Run the equality and collection examples locally.
2. Refactor primitive IDs into strongly typed IDs.
3. Refactor a subscription model from bool flags to explicit states.
4. Compare internal domain state with an explicit JSON contract.
5. Inspect dictionary key behavior with mutable and immutable keys.
